# 人物索引

> 大拿不是天生的。研究那些真正优秀的人，我们才能看见「可复制的路径」。

本索引按**可学习维度**组织，而非简单按国籍或领域排序。每个人的条目都包含：为什么值得学 + 核心可学习点 + 代表作品。

---

## 推荐阅读顺序（新手建议）

1. **李运华** → 最系统的方法论，适合 3-7 年突破瓶颈期
2. **左耳朵耗子** → 深度技术人的长期主义与思考框架
3. **Rich Hickey** → 软件复杂度哲学与第一性原理（强烈推荐）
4. **陈硕** → C++ 系统编程与高质量工程实践
5. **丁奇** → 极致垂直深度的 T 型专家路径
6. **尤雨溪** → 开源影响力与现代前端演进
7. **阮一峰 / 廖雪峰 / 刘大拿** → 教育型长期输出不同风格
8. **Kent Beck / Martin Fowler** → 软件开发心法与架构思想
9. **John Carmack / Jeff Dean** → 全球极致专注与系统设计典范
10. **冯大辉** → 个人品牌与独立思考路径

## 中国大拿

### 李运华（华仔）

- **为什么值得学**：把「如何成为技术大牛」这件事本身，研究得最系统、最可操作的前阿里 P9 专家。他的文章被无数人奉为 3-7 年成长「圣经」。
- **核心可学习点**：Do More / Do Better / 刻意练习；全链路视角；输出倒逼输入；区分「业务代码」与「技术代码」
- **关键身份**：阿里资深技术专家，《从 0 开始学架构》作者

## 超级个体型（新增重点）

**超级个体**指通过个人品牌、内容、开源、独立产品等方式实现高度自主和高价值输出的技术人。他们往往不依赖大公司光环，而是靠自身能力与持续输出建立影响力与收入。

## 如何添加新人物？

1. 在对应子目录创建 `姓名.md`
2. 在本索引中增加条目（格式统一）
3. 在 `索引/全部笔记.md` 中更新
4. 补充至少 3 处与其他笔记的 `[[wikilinks]]`

**人物选择 checklist**：
- [ ] 有公开、可信的成长故事或访谈
- [ ] 存在明显可学习的独特心法或路径
- [ ] 不是纯营销/网红型人物
- [ ] 其经验对 3-8 年技术人有实际指导意义
