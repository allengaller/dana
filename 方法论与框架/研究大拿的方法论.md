# 研究大拿的方法论

> **研究大拿不是为了崇拜，而是为了可复制的成长路径。**

本方法论是 dana 项目研究技术大拿的核心框架，旨在帮助我们系统、深刻、批判性地学习那些真正优秀的工程师。

---

## 核心原则

### 1. 可复制性优先
我们不研究“天赋”，只研究**可被普通人通过努力复制的模式**。

- 关注决策过程、习惯、心法、转折点
- 忽略“运气”“出身”“时代红利”等不可控因素

### 2. 批判性学习
大拿也不是完美的。我们要提取其**优势**，同时识别其**局限**。

- 每个大拿都有时代和个人背景的局限
- 不要全盘接受，要“取其精华，去其糟粕”

### 3. 多维度研究
单一来源容易产生偏见。我们尽量从多个角度交叉验证：

- 本人写作/演讲/访谈
- 同事/下属/合作者的评价
- 公开项目/代码/产品
- 长期观察（而非单次爆款内容）

### 4. 输出倒逼研究
研究必须以输出为目标。光看不写 = 没研究。

推荐研究流程：
1. 收集原始材料
2. 提炼 3-7 个核心可学习点
3. 写成结构化文章（见人物志模板）
4. 加入 dana 知识库

---

## 研究框架（五层模型）

对每位大拿，我们尝试回答以下问题：

### 第一层：身份与时代背景
- 他/她处于什么时代？什么行业阶段？
- 核心优势是什么？（技术深度 / 产品感 / 影响力 / 执行力 / 哲学思考等）

### 第二层：关键转折点
- 职业生涯中有哪些决定性时刻？
- 这些时刻的决策依据是什么？

### 第三层：核心心法与习惯
- 他/她反复强调的底层原则是什么？
- 日常工作/学习/思考的具体习惯有哪些？

### 第四层：可复制的行动
- 一个普通 3-7 年工程师，今天可以模仿的具体行动是什么？
- 哪些是“必须先有一定基础才能复制”的？

### 第五层：局限与代价
- 他的成功付出了什么代价？
- 在今天的环境下，他的路径还有效吗？需要做哪些调整？

---

## 研究材料优先级

1. **本人长期输出**（最高优先）：博客、书籍、演讲、播客、开源项目
2. **深度访谈**（次高）：长形式、诚实的自我复盘
3. **同事/下属评价**（验证用）
4. **公开项目与代码**（最硬的证据）
5. **二手总结**（参考价值最低，容易失真）

---

## 常见研究陷阱（要避免）

- **成功者偏差**：只看成功故事，忽略失败和代价
- **事后诸葛亮**：用现在的视角美化过去的决策
- **偶像化**：把大拿神化，丧失批判能力
- **路径依赖**：认为“因为他成功了，所以这条路最好”
- **只看结果不看过程**：只关注“成了”，忽略“怎么成的”

---

## 输出标准

研究一篇大拿后，理想输出应包含：

- 引言：为什么研究这个人
- 关键转折（3-5 个）
- 核心可学习点（带落地建议）
- 局限与今天该如何看待
- 推荐阅读材料
- 与 dana 其他内容的关联

---

## 研究伦理

- 尊重事实，不编造故事
- 标注信息来源
- 允许大拿有“人设之外”的复杂性
- 研究是为了帮助更多人成长，而不是制造新的偶像

## 真实研究案例示例

### 示例一：研究李运华
**研究重点**：3-7年突破期的可复制方法
**主要材料**：其知乎文章 + 极客时间课程 + 公开访谈
**提炼出的可学习点**：
- Do More / Do Better / 刻意练习
- 全链路视角的重要性
- 输出倒逼输入
**落地行动**：每季度主动认领一个“不是自己负责但有价值”的模块，并写复盘

### 示例二：研究 Rich Hickey
**研究重点**：哲学思考如何影响工程决策
**主要材料**：《Simple Made Easy》演讲 + Clojure 设计文档
**提炼出的可学习点**：
- 简单 vs 容易的区分
- 对状态和时间的建模
- “完成”与“完成工作”的区别
**落地行动**：在下一次架构评审中，主动区分哪些方案是“容易”而不是“简单”

### 示例三：研究程序员鱼皮
**研究重点**：内容创作者如何从 0 到超级个体
**主要材料**：其公众号文章、知识星球分享、公开访谈
**提炼出的可学习点**：
- 垂直深耕 + 持续输出
- 内容 → 社群 → 产品 的变现飞轮
- 个人品牌建设方法
**落地行动**：选择一个垂直领域，每周固定输出 1 篇「项目实战 + 避坑」内容，持续 6 个月

### 示例四：匿名 3-7 年突破案例（框架）
**研究重点**：从“写好业务代码”到“建立技术影响力”的 4 年路径
**主要材料**：匿名工程师复盘（项目驱动学习 + 持续输出 + 刻意练习反馈）
**提炼出的可学习点**：
- 用真实项目作为刻意练习场（而非 side project）
- 输出倒逼输入：每做一个模块就写一篇深度分享
- 主动寻求残酷反馈（开源 PR、内部 review、教学）
**落地行动**：本季度主动认领一个“高风险高价值”项目，并公开复盘 3 次

### 示例五：匿名超级个体路径（框架）
**研究重点**：如何用内容 + 产品实现部分财务自由
**主要材料**：独立开发者匿名分享（博客 + 小工具 + 付费 newsletter）
**提炼出的可学习点**：
- 从解决自己痛点开始产品
- 写作作为长期杠杆
- T 型深 + 梳型广：在一个领域深到别人绕不开，再横向扩展
**落地行动**：本月写一篇“解决我自己痛点”的工具分享文章
