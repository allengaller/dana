# 贡献指南

感谢你愿意为 dana 知识库贡献内容！

## 贡献类型（欢迎）

1. **人物志补充**
   - 新人物的深度复盘（需有清晰可学习点）
   - 已有人物的案例补充或新转折点

2. **真实成长案例**
   - 3-7 年瓶颈突破的匿名复盘（最珍贵）
   - 具体项目驱动成长的故事

3. **资源推荐**
   - 经过自己验证的高质量书籍、播客、工具
   - 必须写明推荐理由

4. **心法与方法论完善**
   - 已有心法的落地案例
   - 新实用框架

## 不接受的内容

- 纯技术教程（请贡献到 CS-Notes 类仓库）
- 鸡汤、成功学、未经思考的笔记
- 低质量流水账式人物介绍

## 提交流程

1. 阅读现有笔记，理解质量标准
2. Fork 仓库 → 新建分支
3. 按照模板格式撰写
4. 提交 PR 并在描述中说明贡献点

## 质量标准（必须满足）

- 具体故事 / 转折点（而非泛泛而谈）
- 至少 3 条可行动的「可学习点」
- 包含「今天可以开始的 1-3 件事」
- 至少 3 处内部 wikilinks
- 语气：深刻、直白、可行动、无鸡汤

## 模板

- 人物志：使用 `模板/人物志模板.md`
- 心法：使用 `模板/心法模板.md`

## 示例贡献

**示例 PR 描述（新增人物 Dario Amodei）**：
- 新增 [[Dario Amodei]] 完整画像：Anthropic 创始人、Constitutional AI 实践、对齐作为规模化优势。
- 包含关键转折（OpenAI 离职、Anthropic 创立、Claude 开发）、5 个可学习点 + 行动、AI 时代特有洞见（宪法式 AI 工程化）。
- 交叉引用 [[AI 对齐的工程实践]]、[[Ilya Sutskever]]、新案例十六。
- 遵循模板：故事 + 转折 + checklist + 局限 + wikilinks。

**示例：匿名成长案例**
- 贡献到研究方法论或成长路径：描述一个普通工程师如何用“项目驱动 + 输出倒逼”在 4 年内从业务代码到技术影响力（不具名）。

## 联系

有问题可直接提 Issue 或在 PR 中讨论。

我们相信：**高质量的内容，比数量更重要**。
