# 必读专栏与博客

> 书籍给你体系，博客给你**鲜活的思考现场**。

好的技术博客不只是知识搬运，而是让你看到一个高手如何**拆解问题、形成判断、持续迭代**。以下是经过严格筛选的高质量持续输出源，按领域分类，覆盖中英文。

---

## 中文技术博客 / 专栏

- **李运华**（极客时间《大厂晋升指南》《从 0 开始学架构》、知乎）
  把架构思维与晋升方法论讲得最系统、最接地气的中文作者。适合 3-7 年阶段反复精读。
  → 搭配阅读 [[李运华]] · [[项目驱动学习法]]

- **左耳朵耗子 / 酷壳 CoolShell**
  已故的陈皓留下了中文技术社区最深刻的一批文章——关于底层原理、程序员成长、技术品味。语气冷静，洞见极深。
  → 搭配阅读 [[左耳朵耗子]] · [[输出倒逼输入]]

- **阮一峰**（科技爱好者周刊、《ES6 标准入门》）
  中文世界最稳定的技术科普源之一。周刊覆盖面广，ES6 教程是前端必读。适合培养「持续跟踪技术趋势」的习惯。
  → 搭配阅读 [[阮一峰]]

- **冯大辉**（早期博客、独立思考）
  从 DBA 到技术管理者，他的博客记录了真实的技术决策与职业取舍，对理解「技术人在组织中的位置」极有价值。
  → 搭配阅读 [[冯大辉]]

- **廖雪峰**（Python / Java / Git 教程）
  中文最流行的编程语言入门系列之一。适合快速查阅基础，也可作为带新人时的推荐资源。
  → 搭配阅读 [[廖雪峰]]

- **丁奇**（数据库技术、《MySQL 技术内幕》系列）
  把 InnoDB 存储引擎讲到极致深度的典范。读他的文章能学会「如何把一个垂直领域研究透」。
  → 搭配阅读 [[丁奇]] · [[系统思维与全链路视角]]

- **云风**（游戏引擎、系统开发）
  国内少有的长期公开分享底层系统开发与游戏引擎设计的大拿。文章硬核、代码扎实，适合对系统编程有兴趣的人。
  → 搭配阅读 [[云风]] · [[源码阅读与造轮子策略]]

- **刘未鹏**（Mind Hacks / 暗时间）
  从认知科学到学习方法，他的博客是中文技术圈最早把「元认知」和「刻意练习」讲清楚的人之一。
  → 搭配阅读 [[刻意练习与及时反馈]]

- **王垠**（编程与语言设计）
  对编程语言设计有极深的思考和犀利的批评。读他的文章能训练你对语言本质的判断力，不必同意，但值得一读。

- **张鑫旭**（CSS / 前端）
  中文前端领域最细致的技术博主之一。文章把 CSS 这类「看似简单实则复杂」的技术讲得极其透彻，是深耕精神的体现。

## 英文技术博客 / 专栏

- **Martin Fowler**（[martinfowler.com](https://martinfowler.com)）
  重构、企业架构模式、持续集成的思想源头。文章清晰、体系化，是理解软件工程最佳实践的核心。
  → 搭配阅读 [[Martin Fowler]]

- **Will Larson**（[lethain.com](https://lethain.com) / [staffeng.com](https://staffeng.com)）
  工程管理领域最重要的当代作者之一。从 Staff Engineer 到技术组织设计，他写出了技术 Leader 需要面对的真实决策。

- **Paul Graham**（[paulgraham.com](http://paulgraham.com)）
  Y Combinator 创始人。他的随笔关于创业、黑客精神和思维方式，影响了整整一代技术人。文字简洁有力，值得反复读。

- **Julia Evans**（[jvns.ca](https://jvns.ca)）
  用漫画和直觉化的方式讲解操作系统、网络、Linux 内核。是把底层知识变得「可读」的典范。

- **Dan McKinley**（[mcfunley.com](https://mcfunley.com)）
  前 Etsy 首席工程师。「Choose Boring Technology」的提出者，关于工程决策取舍的文章极有价值。

- **The Pragmatic Engineer**（[newsletter](https://newsletter.pragmaticengineer.com)）
  当代最有影响力的技术管理 newsletter。覆盖大厂文化、工程管理、架构趋势，信息密度高。

- **High Scalability**（[highscalability.com](http://highscalability.com)）
  专注大规模系统架构案例的博客。想了解顶级公司如何设计系统，这里是最集中的资源。

- **Kent Beck**（[Substack](https://tidyfirst.substack.com)）
  极限编程创始人、TDD 之父。他的 Substack 持续输出关于设计模式、测试策略和代码简洁性的思考。
  → 搭配阅读 [[Kent Beck]]

- **John Carmack**（Facebook posts / .plan files）
  传奇游戏程序员。他的长文和状态更新是工程极致的活教材——关于优化、取舍、技术深度。
  → 搭配阅读 [[John Carmack]]

## 系统设计 / 架构

- **System Design Primer**（[GitHub](https://github.com/donnemartin/system-design-primer)）
  系统设计面试与知识梳理的集大成者。适合 3 年以上工程师系统性地补齐分布式系统认知。

- **ByteByteGo**（[newsletter](https://blog.bytebytego.com)）
  Alex Xu 主理。每期一篇图解系统设计，从消息队列到限流策略，清晰且实用。

- **InfoQ Architecture**（[中文](https://www.infoq.cn) / [英文](https://www.infoq.com)）
  一线架构师的实战分享平台。重点关注「架构师访谈」「人物志」系列，能看到真实大厂的架构决策过程。

## AI / 机器学习

- **Andrej Karpathy**（[blog](https://karpathy.ai) & [YouTube](https://www.youtube.com/@AndrejKarpathy)）
  前 Tesla AI 总监。他的 YouTube 教程是从零理解神经网络的最佳路径，兼具深度与教学天赋。
  → 搭配阅读 [[Andrej Karpathy]]

- **The Batch**（[deeplearning.ai newsletter](https://www.deeplearning.ai/the-batch/)）
  Andrew Ng 团队出品。每周一期 AI 领域要闻精选 + 深度解读，适合保持对行业动态的跟踪。

- **Simon Willison's Weblog**（[simonwillison.net](https://simonwillison.net)）
  当代 LLM 实践领域最活跃的个人博客。每天都在记录和分享 AI 工具使用的真实经验，是理解 AI 应用层的最佳窗口。

- **Chip Huyen's Blog**（[huyenchip.com](https://huyenchip.com)）
  《Designing Machine Learning Systems》作者。关于 MLOps、ML 系统设计、AI 工程化的文章质量极高。

## 阅读建议

1. **不要订而不读**：订阅源控制在 5-8 个以内。如果长期不看，果断取关，保持信息流干净。
2. **精读 + 输出**：每周一篇深度文章就够了。读完写 100-200 字笔记，效果远超泛读十篇。参见 [[输出倒逼输入]]。
3. **按阶段调整重心**：0-3 年侧重基础教程（廖雪峰、阮一峰、Julia Evans）；3-7 年侧重架构与系统设计（李运华、Martin Fowler、System Design Primer）；7 年以上侧重工程领导力（Will Larson、The Pragmatic Engineer）。
4. **中英结合**：英文博客的信息密度和前沿性通常更高，建议至少保持 2-3 个英文源的习惯。
5. **追人不追话题**：好的作者比热门话题更有价值。找到一个与你思维方式共鸣的作者，长期跟踪阅读。
6. **与读书互补**：博客是书的「实时补丁」。读完一本架构书，去关注作者的后续博客，看他的思想如何演进。参见 [[必读书单]]。
