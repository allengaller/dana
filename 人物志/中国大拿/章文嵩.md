# 章文嵩：两周写出 LVS，用开源改变中国互联网基础设施的男人

> **"先迈出第一步。"**
> — 章文嵩，2024 年 CCF 专访中对年轻开发者的寄语

章文嵩可能是中国互联网史上最被低估的"基础设施型"技术人之一：

- 他在读博期间**只用两周**就写出了 LVS（Linux Virtual Server），并完全开源
- 这个项目后来进入 **Linux 官方内核**，成为四层负载均衡的**事实标准**
- 他是中国工程师里**最早向 Linux 内核贡献万行级代码**的人之一
- 他拒绝了 Google、Facebook 的邀请，选择在国内做 CDN、TFS、Tengine、阿里云 ECS/RDS/OSS/SLB 等基础软件
- 他在阿里推动开源委员会，让"开源"成为阿里技术文化的一部分
- 他在 43 岁从阿里云 CTO 转到滴滴负责基础平台，51 岁又开始做云原生与 AI Infra 的创业孵化与投资

我们研究他，不是因为他"成功"，而是因为他展示了**一种中国技术人极少走通的路径**：用一个硬核开源项目，建立全球影响力，然后在不同组织里持续放大这种影响力，最终把自己变成基础设施生态的"放大器"。

**但请注意**：章文嵩的路径依赖特定时代窗口（1998 年的开源萌芽期、2009 年的阿里技术升级期），也经历过创业失败和组织文化摩擦。我们提炼的是**开源与系统建设的可复制方法**，而不是**个人履历的简单模仿**。

---

## 为什么值得学

对 3-7 年技术人而言，章文嵩的独特价值在于：

1. **他证明了"做一个有用的东西"比"追热点"更有长期价值**——LVS 诞生于 1998 年，直到今天仍是全球互联网基础设施的核心组件。一个看似简单的问题（负载均衡），一旦被彻底解决，可以影响二十年。
2. **他展示了"开源"作为全球影响力的杠杆**——在出国交流还不方便的年代，他通过开源代码和邮件列表，与保加利亚的 Julian Anastasov、Linux 内核网络层总管 David Miller 等全球顶尖工程师协作。
3. **他从"个人开源项目"走到"大规模工程体系"**——这填补了"只会写代码"和"只会做管理"之间的鸿沟。他既能写内核模块，也能建设阿里 CDN、双 11 基础平台、滴滴智慧交通云。

## 关键转折

### 转折一：从街机少年到计算机兴趣小组（1987）——兴趣的意外入口

章文嵩 1973 年生于上海。初中时痴迷街机游戏，零花钱全耗在上面。后来一个同学告诉他"有个地方打游戏不要钱"，他周末带一个面包去上海青少年科学指导站，一待就是一天。

在那里，他学会了 Basic、Pascal、6502 汇编，甚至尝试在苹果机上写中文操作系统。高中三年打下的基础，让他 1990 年考入国防科技大学计算机系时"如鱼得水"。

**关键信号**：章文嵩的起点不是"天赋异禀"，而是**游戏驱动的兴趣**。这提醒我们：早期技术热情的入口可以很俗气，关键是能否把它转化为持续的学习。

### 转折二：硕士想开源数据库被拒——学会"从头做一个能开源的东西"

硕士期间，章文嵩花了两年时间写了一个数据库，想开源。导师不同意：这是国家课题，当时没有相关政策支持。程序后来慢慢消亡。

毕业时，他又想把过往课题成果开源，再次被拒。

多次受挫后，章文嵩决定：**与其改造已有成果，不如从头做一个全新的、可以开源的软件**。

**关键信号**：这是章文嵩开源哲学的起点——**所有权和许可比代码本身更重要**。他也因此学会了从"我能做什么"转向"我能独立交付并开放什么"。

### 转折三：两周写出 LVS（1998）——从学术问题到全球基础设施

1998 年，章文嵩在国防科大读博。湖南电信一个系统集成项目需要多台服务器做负载均衡，但一套思科 LocalDirector 要几万美元。章文嵩觉得："这东西并不难写，为什么要卖那么贵？"

1998 年 5 月，他花**两周**写好了 LVS 的代码和使用文档，放在互联网上，并在 Linux Kernel 邮件列表里广而告之。

第一个用户来自澳洲，是一家小型 ISP。他用 LVS 把几台服务器组合起来做负载均衡，既不用花钱又发挥了设备最大功效。

**关键信号**：LVS 的诞生不是宏大计划，而是**解决一个具体、真实、被高价商业产品垄断的问题**。开源让它获得了指数级反馈——每天都有全球用户发来建议和补丁。

### 转折四：LVS 进入 Linux 官方内核（2003）——开源社区的真正认可

2003 年，章文嵩意外收到 Linux 内核网络层总管 David Miller 的邮件："LVS 代码还没进官方内核吗？"

章文嵩回复"还没"，当晚 David Miller 就把它加了进去。

从此，LVS 代码从 Linux 2.4 起一直在官方内核中。全世界约有十万套 LVS 集群在运行，阿里、腾讯、百度、新浪、谷歌都在使用。商用负载均衡 F5 的中国总代理曾对章文嵩说："F5 在中国最大的竞争对手就是你的 LVS。"

**关键信号**：真正的开源影响力不是 GitHub star，而是**被基础设施默认采用**。David Miller 的一句话，比任何奖项都更有分量。

### 转折五：从阿里到滴滴（2009-2016）——把开源能力转化为组织级工程体系

2009 年，章文嵩离开创业低谷，加入阿里巴巴。阿里给了他团队和资源，让他专注核心系统研发：CDN、TFS、Tengine、数据库、内核、网络、图像搜索、大数据、云计算。

- 他带领团队建设的 CDN 为淘宝节约了 **20 多亿元**建设成本
- 交易处理速度从每秒 3000 笔提升到每秒 10 万笔以上
- 他推动阿里开源委员会，让开源成为阿里工程师文化的一部分
- 他历任淘宝资深技术总监、淘宝技术委员会主席、阿里副总裁、阿里云 CTO/首席科学家

2016 年，43 岁的章文嵩加入滴滴，任高级副总裁、基础平台负责人、工程技术委员会主席。这一步意味着从"电商基础设施"转向"出行基础设施"。

**关键信号**：章文嵩的职业生涯展示了**技术影响力的三次跃迁**：
1. 个人开源项目（LVS）
2. 公司级基础设施平台（阿里 CDN/云计算）
3. 行业级基础设施生态（滴滴智慧交通、CCF 开源委员会）

每次跃迁都需要重新学习组织语言，但也把个人技术品牌不断放大。

### 转折六：从一线技术人到投资人与生态培育者（2021-）

2021 年，章文嵩离开滴滴，加入高瓴任运营合伙人，负责技术方向布局和人才引进。2024 年，他获 CCF 卓越服务奖，并创办安托盟丘（杭州）科技有限公司，主要从事云原生和 AI Infra 的创业孵化与投资。

**关键信号**：章文嵩的晚年角色从"建造者"变为"培育者"。这是技术影响力的高级形态——**不是自己做所有事，而是识别和支持下一个能做出基础设施的人**。

## 核心心法与习惯

### 1. "先迈出第一步"——行动主义者的起点

章文嵩在多次采访中强调：想做一件事，先迈出第一步。

- LVS 不是计划出来的，是"两周写出来"的
- 开源不是等完美了再开放，是"写好了就放上去"
- 他不会先考虑商业模式，而是先考虑"这个东西有没有用"

**可学习的**：很多技术人卡在"准备阶段"——等学更多、等更完善、等更好的时机。章文嵩的反直觉做法是：**用最小可运行版本先获得真实反馈，再迭代**。

### 2. "做一个有用的人"——工程价值的朴素标准

章文嵩对自己的定位一直很朴素："一个比较注重做实际工作的有用的人。"

- 他不追求论文数量，而是看系统是否被使用
- 他不追求技术炫技，而是看是否解决真实成本问题
- 他在阿里推动"低碳数据中心"，把淘宝单笔交易能耗从"煮四个鸡蛋"降到"煮一个鸡蛋"

**可学习的**：把"有用"作为第一性标准，可以避免很多技术人的通病——**为了技术而技术**。

### 3. 开源作为"反馈放大器"

章文嵩的开源方法论：

- **完全开源，不搞双版本**：他拒绝"开源一个版本、闭源卖一个版本"，因为"那多麻烦"
- **给每个贡献者署名**：他认为每个补丁贡献者都值得被署名，这是社区荣誉
- **用邮件列表做协作**：在 GitHub 出现前，他就通过 Linux Kernel 邮件列表与全球开发者协作
- **接受真实场景反馈**：澳洲 ISP、瑞士用户寄来的饼干、保加利亚开发者 Julian Anastasov 的长期贡献，都说明 LVS 的影响力来自真实使用

**可学习的**：开源不是"把代码放出来"，而是**建立一套持续反馈和协作的机制**。

### 4. 系统建设走在业务前面

章文嵩在淘宝时发现："系统建设最好走在前面，不能等网站瘫痪后再亡羊补牢。"

2010 年玉树地震时，全国网站首页变黑白，淘宝流量暴增数倍。章文嵩带着团队半年前建设的 CDN 扛住了冲击。刘振飞回忆：章文嵩计算后说"应该能扛住，不用回"，结果确实扛住了。

**可学习的**：基础设施的价值往往在危机时刻才显现。平时做冗余、做优化、做容量规划，看起来"浪费"，其实是在买期权。

### 5. 技术人才的三种分类

章文嵩在淘宝招聘时，把需要的人才分为三类：

1. **对大规模系统有经验的人**——知道什么会坏、怎么扩展
2. **追求极致的技术型人才**——能把性能、成本压榨到极限
3. **有经济思维的技术人才**——懂得用技术创造商业价值，而不只是解决技术问题

**可学习的**：技术团队不是只有一种"牛人"。章文嵩的用人观提醒我们：**成熟的技术组织需要能力互补，而不是全招同一种天才**。

## 可复制的行动（普通人今天就能开始）

### 🔵 今天可以开始的 3 件事

1. **找到一个"高价且可被替代"的小问题**
   - 观察你工作中有没有某个工具/流程又贵又难用
   - 问自己：如果我用开源方式重写一个最小版本，会有人用吗？
   - 用周末两周写一个可运行的原型

2. **给正在使用的开源项目提交一个真实补丁**
   - 不要从"重写一个著名项目"开始
   - 选一个你每天都在用的工具/库/框架
   - 修复一个小 bug、改进一段文档、补充一个测试
   - 关键是**迈出协作的第一步**

3. **用"成本意识"审视你负责的系统**
   - 选一个你维护的服务/模块
   - 计算：单次请求消耗多少 CPU/内存/存储？
   - 找出一个能把成本降低 10% 以上的优化点
   - 用数据说服团队

### 🟡 本季度可以做的事

- **建立你的"两周项目"清单**：章文嵩能用两周写出 LVS 的核心，不是因为他时间多，而是因为他能聚焦最小核心。列出 3 个你能在两周内完成并开源/分享的小项目。
- **参与一个技术社区的真实协作**：邮件列表、GitHub Issue、Discord/Slack 讨论都可以。目标是学会"给别人反馈"和"接受别人反馈"。
- **写一份"系统健康度"报告**：像章文嵩做双 11 容量规划一样，给你负责的系统做一份容量、成本、风险三维度评估。

## 局限与代价

### 1. 时代窗口的不可复制性

LVS 诞生于 1998 年，中国互联网刚起步，Linux 生态正在扩张，负载均衡领域几乎空白。今天的技术人很难再找到"一个空白领域 + 一个成熟平台 + 一个未被满足需求"的交汇点。

**启示**：与其复制"写出一个 LVS"，不如学习他**识别高价值基础设施问题**的眼光。

### 2. 开源不等于商业模式

章文嵩本人多次创业，但并非都成功。TelTel、ChinaCluster 等项目说明：**技术影响力可以很大，商业成功却是另一套逻辑**。

**启示**：如果你选择开源路线，要清楚自己是在积累影响力、构建生态，还是直接变现。三者路径不同。

### 3. "算法比围棋复杂 100 倍"争议的教训

2018 年，章文嵩在 REMIX 大会上把滴滴派单算法与 AlphaGo 对比，说"滴滴的算法比围棋复杂 100 倍"。这句话引起不小争议。

**批判性视角**：技术人容易低估自己领域外问题的难度，也容易用"复杂度"作为价值判断。更成熟的表达是：**每个领域有其独特复杂度，跨领域比较往往失焦**。

### 4. 组织文化适应成本

从国防科大到创业公司，到阿里，到滴滴，再到高瓴、自主创业，章文嵩经历了多次组织文化切换。每次切换都需要放弃一部分舒适区。

**启示**：技术影响力可以让你跨组织流动，但**组织语言、管理方法、商业逻辑需要重新学习**。

## 与 dana 其他内容的关联

- **同为基础设施创造者**：[[Ken Thompson]] 用 Unix 和 Go 定义了操作系统与编程语言基础设施，章文嵩用 LVS 定义了中国互联网负载均衡基础设施。两人都是"做少而精的硬核系统"的代表。
- **与开源心法呼应**：章文嵩的实践是 [[源码阅读与造轮子策略]] 的反向——不是先读源码再造轮子，而是**先造一个最小轮子，再让它被源码级社区接受**。
- **系统思维**：他在阿里和滴滴做 CDN、容量规划、低碳优化的经历，是 [[系统思维与全链路视角]] 在超大规模工程中的体现。
- **做难事**：从博士期间单枪匹马写 LVS，到 43 岁从阿里云 CTO 转岗滴滴，都是 [[做难事：突破舒适区]] 的真实案例。
- **与李运华的对比**：[[李运华]] 研究"如何成为技术大牛"的方法论，章文嵩则用一生实践了"用硬核系统建立影响力"的路径。一个讲方法，一个做示范。
