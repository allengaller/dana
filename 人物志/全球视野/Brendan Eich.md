# Brendan Eich：10 天发明 JavaScript，半生守护开放 Web 的男人

> **"在线广告演变成了一个掠夺用户注意力价值的监控系统。Brave 要做的是把用户放在第一位，消除代理人与委托人之间的利益冲突。"**
> — Brendan Eich，Brave Software CEO

Brendan Eich 是互联网史上**最具争议的创造者**之一：

- 他在 1995 年用 **10 天**为 Netscape 浏览器写出了 JavaScript 原型
- 他创造的这门语言，后来成为与 HTML、CSS 并列的**Web 三大核心技术**之一
- 他是 **Mozilla 联合创始人**，带领 Firefox 挑战微软 Internet Explorer 的垄断
- 他在 2014 年成为 Mozilla CEO，又因 2008 年一笔反对同性婚姻的政治捐款，**上任 11 天后被迫辞职**
- 他在 2015 年创办 **Brave Software**，用隐私浏览器和 Basic Attention Token（BAT）挑战 Google/Facebook 的广告模式
- 截至 2025 年，Brave 浏览器月活用户超过 **8700 万**

我们研究他，不是因为他"成功"，而是因为他展示了**一种罕见的职业轨迹**：从一个语言设计者，到一个开源运动组织者，再到一个隐私倡导者和创业者——每次转型都伴随着巨大争议，但他始终没有离开"开放 Web"这条主线。

**但请注意**：JavaScript 的诞生充满妥协与仓促，它的很多设计缺陷（类型强制、全局变量、this 绑定等）至今困扰着开发者；Brave 的 BAT 模式也饱受"加密货币炒作"和"广告替换"的质疑。我们提炼的是**在约束下创造、在争议中坚持**的方法，而不是简单崇拜。

---

## 为什么值得学

对 3-7 年技术人而言，Brendan Eich 的独特价值在于：

1. **他证明了"足够好的原型"比"完美的设计"更能赢得时间**——JavaScript 是 10 天写出来的"应急方案"，但它通过快速迭代和标准化，最终统治了 Web 客户端。这提醒我们：在激烈竞争的市场中，**时机和迭代速度有时比初始质量更重要**。
2. **他展示了技术人如何穿越多次范式转移**——从客户端脚本语言（JavaScript），到开源浏览器（Mozilla/Firefox），再到隐私浏览器与代币经济（Brave/BAT）。每一次他都抓住了 Web 生态的一个核心矛盾。
3. **他把"用户主权"作为持续 30 年的技术伦理主线**——无论是 JavaScript 让网页可交互、Firefox 对抗 IE 垄断，还是 Brave 阻止追踪广告，他的目标始终是**把控制权从平台夺回给用户**。
4. **他的失败与反弹同样值得学习**——Mozilla CEO 事件是一个关于"公众人物、政治观点与组织使命冲突"的极端案例。他后续创办 Brave 的方式，展示了如何在职业低谷后重新出发。

## 关键转折

### 转折一：从数学到系统编程（1980s）——扎实基础的时期

Brendan Eich 1961 年出生于宾夕法尼亚州匹兹堡。他在 Santa Clara University 获得数学与计算机科学学士学位，在伊利诺伊大学厄巴纳-香槟分校（UIUC）获得硕士学位。

早期他在 **Silicon Graphics** 从事操作系统开发，之后加入 **MicroUnity Systems Engineering**。这段经历让他接触到底层系统编程、网络和硬件交互，为他后来设计浏览器语言打下了系统级基础。

**关键信号**：Eich 不是从 Web 前端起步的，而是从**操作系统和底层系统**开始。这解释了为什么他能在浏览器层思考问题——他知道软件栈的每一层如何交互。

---

### 转折二：10 天发明 JavaScript（1995）——在约束下创造的历史性妥协

1995 年 4 月，Eich 加入 Netscape。他的初始任务是为浏览器添加 Scheme（一种 Lisp 方言）。但 Netscape 当时正与 Sun Microsystems 谈判合作，希望让 Netscape 浏览器支持 Java。管理层突然改变方向，要求新语言必须**看起来像 Java**。

为了赶上 Netscape Navigator 2.0 Beta 的发布节点，Eich 在**10 天内**完成了第一个原型，内部代号 **Mocha**。随后改名为 **LiveScript**，最后在 1995 年 12 月为了营销需要改名为 **JavaScript**（与 Java 其实没有直接关系）。

JavaScript 的设计深受 **Scheme**（一等函数、闭包）和 **Self**（基于原型的对象系统）影响，但外观被迫模仿 Java。这种内在张力造就了 JavaScript 后来的很多特点与争议。

Eich 后来回忆："那 10 天我没睡够。"之后几个月，团队疯狂修复安全漏洞，比如引用计数导致的内存安全问题。

**关键信号**：JavaScript 不是理想设计的产物，而是**商业压力、技术审美和截止日期共同作用的结果**。这提醒我们：**伟大产品的第一个版本往往是粗糙的，关键是能否持续迭代和标准化**。

---

### 转折三：Mozilla 与 Firefox（1998-2005）——从闭源失败中抢救开源火种

1998 年，Netscape 被 AOL 收购后，Eich 与 Mitchell Baker、Jamie Zawinski 等人共同创立 **Mozilla 项目**，初衷是管理 Netscape 源代码的开源贡献。

2003 年，AOL 关闭 Netscape 浏览器部门。Eich 帮助将 Mozilla 项目独立为 **Mozilla 基金会**，后来又成立 **Mozilla 公司**作为营利分支。2005 年，他成为 Mozilla 公司 CTO，主导 Firefox 浏览器和 SpiderMonkey JavaScript 引擎的发展。

Firefox 在 2000 年代中期成功挑战了 Internet Explorer 的垄断地位，推动了 Web 标准的复兴。Eich 在此期间也深度参与 **ECMAScript** 标准化工作，确保 JavaScript 不会分裂为多个不兼容的方言。

**关键信号**：Mozilla 项目展示了**如何在商业失败中抢救开源资产**。Netscape 作为公司失败了，但它的代码通过开源变成了一把对抗垄断的武器。

---

### 转折四：Mozilla CEO 与 Prop 8 争议（2014）——技术成就与个人政治的碰撞

2014 年 3 月 24 日，Eich 被任命为 Mozilla CEO。但一周后，媒体披露他在 2008 年向加州 **Proposition 8**（一项禁止同性婚姻的法案）捐款 1000 美元。

这场风波迅速升级：OkCupid 公开呼吁 Firefox 用户改用其他浏览器，Mozilla 员工和开源社区出现强烈抗议。2014 年 4 月 3 日，Eich 宣布辞职。

他在辞职声明中说："Mozilla 的使命比我们任何人都重要。在当前情况下，我无法成为有效的领导者。"

**关键信号**：这是科技史上一个标志性的**"取消文化"案例**。它提醒我们：
- 技术人的公共身份越来越难以与私人政治完全分离
- 组织使命与个人价值观冲突时，可能需要做出痛苦选择
- 一次多年前的政治捐款，可以对职业生涯造成巨大影响

---

### 转折五：创办 Brave Software（2015）——从低谷重新出发

离开 Mozilla 后，Eich 没有退出科技行业。2015 年 5 月，他与前 Mozilla 工程师 Brian Bondy 共同创立 **Brave Software**。

Brave 浏览器的核心主张是：
- 默认阻止广告和第三方追踪器
- 通过内置 Shields 提供隐私保护
- 引入 **Basic Attention Token（BAT）**，让用户选择观看隐私尊重型广告并获得代币奖励
- 广告主可以直接触达用户，而不需要 Google/Facebook 式的数据监控

2016 年 1 月，Brave 发布开发者预览版。2019 年，Brave Rewards 正式上线。到 2025 年，Brave 月活用户达到 8700 万，日活约 3600 万。

**关键信号**：Eich 在 50 多岁时选择从头创业，瞄准的是**自己曾经参与构建的 Web 广告系统**。这是一种罕见的"自我修正"——他承认 JavaScript 和 cookies 在 1990 年代为今天的监控资本主义埋下了种子，现在他想改变这个系统。

## 核心心法与习惯

### 1. "先做出来"——行动优先于完美

JavaScript 的 10 天传奇常被误读为" rushed hack"，但 Eich 的核心方法论是：**在 deadline 和市场压力下，先做出可运行的东西，再持续迭代**。

他后来谈到："好的想法很多，好的执行很少。时机、想法和执行的混合至关重要。"

**可学习的**：
- 面对紧迫任务时，先定义最小可运行版本
- 不要纠结于第一版是否优雅，要让它能被真实用户使用
- 留下迭代空间：JavaScript 最初粗糙，但 ECMAScript 标准让它有了持续进化的框架

---

### 2. 用标准化对抗分裂

浏览器大战时期，微软推出 JScript，各家浏览器实现不兼容，Web 开发者苦不堪言。Eich 推动 JavaScript 进入 **ECMA International** 标准化流程，1997 年发布 ECMAScript 第一版。

虽然标准化过程缓慢且充满政治博弈，但它最终避免了 JavaScript 像当年 HTML 一样分裂成多个方言。

**可学习的**：
- 如果你创造了一个被广泛使用的工具/协议，**标准比实现更重要**
- 标准化可以保护用户和开发者免受厂商锁定的伤害
- 参与标准组织是技术影响力的高级形式

---

### 3. 把用户主权作为设计第一性

从 JavaScript 到 Firefox 再到 Brave，Eich 的设计始终围绕一个原则：**把控制权交给用户**。

- JavaScript 让网页从静态文档变成可交互应用
- Firefox 让用户有 IE 之外的开放选择
- Brave 默认阻止追踪，让用户控制自己的注意力和数据

**可学习的**：
- 设计产品时，先问"这增强了用户自主权，还是平台控制权？"
- 在免费服务时代，"用户第一"往往意味着**主动限制自己的商业模式**
- 隐私和便利不是零和，但需要刻意设计

---

### 4. 在失败中保持方向感

Mozilla CEO 事件是 Eich 职业生涯的重大挫折，但他没有因此放弃对开放 Web 的承诺。相反，他把这次经历转化为创办 Brave 的动力。

他在 InfoWorld 采访中直言："我责怪自己发明了 JavaScript，但我也看到它助长了一个掠夺用户的广告系统。Brave 就是要修复这个问题。"

**可学习的**：
- 职业生涯的低谷不一定意味着方向的错误
- 如果你的核心使命依然成立，换一家公司、换一种形式，可以继续推进
- 公开失败后的反弹，需要比顺境更强的内在动机

---

### 5. 质疑每一个"默认设置"

Brave 的核心理念之一是：**隐私应该是默认的，而不是需要用户手动开启的选项**。

Eich 批评 Chrome 默认追踪用户、默认允许第三方 cookies、默认加载大量追踪脚本。他认为，普通用户不应该成为隐私专家才能安全上网。

**可学习的**：
- 在设计系统时，默认设置比选项说明更有影响力
- "默认安全/默认隐私"是一种强大的产品设计伦理
- 但也要注意：默认设置会影响商业模式，需要勇气和取舍

## 可复制的行动（普通人今天就能开始）

### 🔵 今天可以开始的 3 件事

1. **学习 JavaScript 的设计历史，理解它为什么是今天这个样子**
   - 读 Eich 在 JSConf 或 Lex Fridman 播客中关于 JavaScript 起源的访谈
   - 重点理解：Scheme 的影响、Self 的影响、为什么它看起来像 Java
   - 这会让你对"语言设计妥协"有更现实的认知

2. **用 Brave 或 Firefox 一周，体验"隐私默认"的设计差异**
   - 观察哪些网站加载变快、哪些功能变卡
   - 记录你遇到的兼容性问题和心理感受
   - 思考：隐私保护的产品，如何在便利性和安全性之间取舍

3. **为 ECMAScript 或你使用的一个 Web 标准提交反馈**
   - 不必是代码贡献，可以是一个 issue、文档建议或 use case
   - 了解标准化组织如何收集社区反馈
   - 体会"标准制定是政治+技术"的真实过程

### 🟡 本季度可以做的事

- **设计一个"用户主权优先"的小功能**：在你负责的产品中，找出一个可以默认保护用户隐私/时间的功能（比如默认不追踪、默认最小权限）
- **研究一个 Web 标准的演进史**：选一个你常用的 API（Fetch、Promise、Web Workers 等），读它的提案和讨论记录
- **写一份"JavaScript 设计取舍"学习笔记**：从 Eich 的访谈出发，总结 3 个你认为是正确取舍、3 个你认为是历史包袱的设计点

## 局限与代价

### 1. JavaScript 的设计债务

JavaScript 的 10 天创建过程留下了大量长期问题：
- 类型强制（type coercion）导致大量意外行为
- `this` 绑定的复杂性
- 全局变量和作用域提升（hoisting）
- 早期缺乏模块系统

虽然 ES6 及以后的版本修复了很多问题，但这些历史包袱至今仍在。

**启示**：快速原型可以赢得市场，但**设计债务可能需要 20 年才能还清**。在做技术决策时，要区分"可以以后修复的"和"会锁住未来方向的"。

### 2. Prop 8 争议的持久影响

Eich 因个人政治捐款被迫离开 Mozilla，这件事对科技行业影响深远。一方面，它显示了公众人物难以与私人行为完全分离；另一方面，也引发了关于"政治正确是否过度"的争论。

**批判性视角**：一个 CEO 的个人价值观是否应该与组织使命完全绑定？这个问题没有标准答案，但 Eich 的案例提醒我们：**在技术行业，公共身份本身就是一种政治资产/负债**。

### 3. Brave/BAT 模式的争议

Brave 和 BAT 虽然获得了大量用户，但也面临批评：
- **广告替换质疑**：Brave 阻止网站原有广告，再展示自己的广告，被批评为"用新垄断替代旧垄断"
- **代币经济风险**：BAT 作为加密货币，价格波动大，且代币分配高度集中
- **Chromium 依赖**：Brave 基于 Chromium，仍间接支持 Google 的 Blink 引擎垄断
- **Telemetry 默认开启**：与隐私优先的形象有一定冲突

**启示**："用户主权"是很好的叙事，但具体商业模式中往往充满利益冲突。批判性接受任何"颠覆者"的自我表述。

### 4. 从建造者到 CEO 的角色张力

Eich 在 Brave 的角色不仅是首席工程师，还是 CEO。他公开承认自己不爱管理事务，更享受编码和架构。但 CEO 职责要求他处理融资、团队、合规、公关等非技术事务。

**启示**：技术型创始人常常高估自己对管理工作的容忍度。如果你选择创业，要么找到互补合伙人，要么准备好学习一套完全不同的技能。

## 代表作品

- **JavaScript / ECMAScript**：1995 年创造，Web 三大核心技术之一，全球最广泛使用的编程语言之一
- **SpiderMonkey**：Netscape/Mozilla 的 JavaScript 引擎，Firefox 的核心组件
- **Mozilla / Firefox**：1998 年共同创立，推动开源浏览器复兴，打破 IE 垄断
- **Brave 浏览器**：2015 年共同创立，隐私优先浏览器，月活用户超过 8700 万（2025）
- **Basic Attention Token（BAT）**：基于以太坊的注意力代币，试图重构数字广告经济

## 与 dana 其他内容的关联

- **与 Tim Berners-Lee 互补**：[[Tim Berners-Lee]] 发明了 Web 的协议与文档结构（HTML/HTTP/URL），Brendan Eich 发明了让 Web 活起来的编程语言。两人共同定义了现代 Web 的骨骼与神经系统。
- **与 Rob Pike 的对比**：[[Rob Pike]] 设计 Go 语言时强调"清晰优于聪明"和软件工程导向；Eich 设计 JavaScript 时则在 Scheme 的优雅与 Java 的市场压力之间走钢丝。两者展示了语言设计的两种极端情境。
- **与 Rich Hickey 的对比**：[[Rich Hickey]] 用 Clojure 展示"简单 vs 容易"的区分；Eich 则用 JavaScript 展示了"容易但不简单"的语言如何在现实中获胜。
- **与尤雨溪的呼应**：[[尤雨溪]] 用 Vue.js 降低前端开发门槛，延续了 Eich 开启的"让 Web 可编程"的方向。
- **与系统思维的呼应**：Brave 试图重构广告系统的激励机制，是 [[系统思维与全链路视角]] 在商业生态设计中的应用。
- **与做难事的呼应**：从 Mozilla CEO 风波中重新创办 Brave，是 [[做难事：突破舒适区]] 的典型案例。
- **与影响力公式的呼应**：Eich 用一项核心技术（JavaScript）+ 持续 30 年的开源参与 + 多次关键节点发声，诠释了 [[影响力 = 深度 × 持续输出 × 连接]]。
